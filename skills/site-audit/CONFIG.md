# Site Audit Configuration

**Copy this file to new projects and fill in the blanks.** Every setting marked `REQUIRED` must be set before first run.

## REQUIRED — Site Identity

| Setting | Value | Description |
|---------|-------|-------------|
| `base_url` | **REQUIRED** | Site base URL, e.g. `https://mysite.local` |
| `wp_root` | `.` | Path to WordPress root (relative or absolute) |

## REQUIRED — URL Scope

| Setting | Value | Description |
|---------|-------|-------------|
| `url_source` | `sitemap` | `sitemap`, `file:<path>`, or comma-separated list |
| `max_urls` | **REQUIRED** | Cap URLs to crawl — set based on site size (50, 200, 500) |

No default `max_urls` — you choose. Small site: 50. Medium: 200. Large: 500+.

## REQUIRED — Fix Loop Iteration Cap

| Setting | Value | Description |
|---------|-------|-------------|
| `max_iterations` | **REQUIRED** | Max fix loop iterations — recommend 2x `max_urls` |
| `max_fix_attempts` | 3 | Attempts per error before skipping |

## Thresholds (RED)

Tune these to your site's performance profile. A WooCommerce site with 200 queries might be normal. A brochure site with 50 might be bloated.

| Metric | Default | Notes |
|--------|---------|-------|
| `query_count` | > 100 | Total DB queries per page |
| `duplicate_queries` | > 5 | Duplicate query patterns |
| `slow_query_ms` | > 50 | Any single query over this (0.05s) |
| `ttfb_ms` | > 500 | Time to first byte |
| `load_ms` | > 3000 | Full page load |
| `memory_mb` | > 64 | PHP memory usage |
| `peak_memory_mb` | > 128 | Peak PHP memory |
| `php_errors` | > 0 | Any PHP error/warning/notice |
| `js_errors` | > 0 | Any JS console error or exception |

AMBER: within 70% of RED threshold.

Override via `thresholds.json`:
```json
{
  "query_count": 150,
  "slow_query_ms": 100
}
```
Pass with: `python3 audit.py --thresholds thresholds.json`

## Log Paths (auto-detected, override if different)

| Source | Default path | Notes |
|--------|-------------|-------|
| QM JSON output | `wp-content/qm-output/*.json` | Core data source — must exist |
| JS errors | `wp-content/qm-output/js-errors.json` | Generated by Playwright crawl |
| PHP debug log | `wp-content/debug.log` | Requires WP_DEBUG_LOG enabled |
| PHP error log | `logs/php-error.log` or `wp-content/logs/php-error-logs.csv` | Text or CSV — both auto-detected |
| Access log | `logs/access.log` or `wp-content/logs/access-logs.csv`       | Text or CSV — both auto-detected |

Override any path in `audit.py` with `--qm-dir`, `--debug-log`, `--php-log`, `--access-log`.

## QM Auth Cookie

```
QM_COOKIE = { name: 'qm_auth', value: '1' }
```

Value must match whatever the QM mu-plugin checks. Any truthy value works with the default mu-plugin.

## Pre-Audit Checklist

- [ ] Caching plugins disabled (W3 Total Cache, WP Rocket, etc.)
- [ ] `QM_DB_EXPENSIVE=0.05` or your threshold in wp-config.php
- [ ] QM JSON dispatcher mu-plugin active
- [ ] `wp-content/qm-output/` directory exists and is writable
- [ ] Sitemap accessible (WordPress default: `/wp-sitemap.xml`)
- [ ] `QM_COOKIE` value in mu-plugin matches crawl cookie value
- [ ] Playwright installed: `cd tests/playwright && npm install`

## Playwright Crawl Config

Edit `tests/playwright/src/crawl.spec.ts` before running:
- `BASE`: your site URL (must match `base_url` above)
- `MAX_URLS`: pull from `max_urls` setting above
- `QM_COOKIE`: must match mu-plugin

See `references/playwright-crawl.md` for setup instructions.
